# 质量与维护

这一部分记录网站内容的校对、参考资料和后续维护事项。它不是知识主线，但对保证网页长期可用很重要。

## 当前页面

1. [内容核查记录](content-review.md)：记录已经检查过的页面、已修复的问题和仍需复核的内容。
2. [参考资料](../references.md)：集中放通用参考文献和后续可补充的来源。

## 维护重点

| 项目 | 检查内容 |
| --- | --- |
| 渲染 | Markdown 表格、代码块、公式和图片是否正常显示 |
| 内容 | 概念、公式、术语和例子是否准确 |
| 导航 | 新增页面是否进入 `mkdocs.yml`，是否能从侧边栏访问 |
| 结构 | 单页是否过长，是否需要拆分成更小章节 |
| 参考 | 论文、书籍、软件文档是否补齐出处 |

## 后续规则

- 新增专题时先建概览页，再逐步补子页面。
- 每次移动页面后检查站内链接。
- 每次提交前运行 `mkdocs build --strict`。
- 公式、表格和代码块修改后要实际渲染检查。
